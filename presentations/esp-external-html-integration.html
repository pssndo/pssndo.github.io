ESP External HTML Integration

Research on Email Service Providers that support fetching external HTML content at send time

What Is This?

Many Email Service Providers (ESPs) support a powerful feature: fetching HTML content from an external URL at the moment an email is sent. This enables dynamic ad insertion without requiring publishers to update their email templates.

How It Works

1

Publisher adds a placeholder in their ESP template

2

Publisher configures the ad server URL

3

Ad Server returns ready HTML with the ad

4

ESP replaces placeholder with ad content

Benefits

⚡

Minimal Technical Effort

Publishers just add a placeholder and URL - no complex integration needed

🔄

Dynamic Content

Each email can display a different ad without template changes

🎯

Personalization Ready

Pass user ID, segment, or other parameters via URL for targeted ads

🛠️

Zero Maintenance

No template updates needed when creatives change

Implementation Example

URL configured by publisher:

https://ads.example.com/placement/12345?user={email}&segment={profile.segment}

Ad server response:

<a href="https://track.example.com/click/xyz">
  <img src="https://cdn.example.com/creative/abc.jpg" alt="Ad" />
</a>

Note: The ad server must return HTML with Content-Type: text/html header.

⚠️ Partial Support

These ESPs support some form of external content, typically limited to RSS feeds or structured data formats.

ESP Limitations
Mailchimp RSS-to-Email only; import from URL at design time, not arbitrary HTML fetch
Listrak Content URL tags available in HTML editor only
Mailup RSS/Atom feeds; no JavaScript; only BODY content used
Sendlane RSS feeds only, not arbitrary HTML URLs
Webpower / Spotler RSS feed integration only
ActiveTrail RSS-to-Email campaigns only
Agillic Dynamic URLs for images/links, not full HTML content
Click Dimensions Metadata extraction only, not full HTML rendering
Useinsider API call integration in workflows, no dedicated external fetch block
Zeta Global Zeta Markup Language + API; no documented send-time URL fetch
Selligent Dynamic HTML via API, not URL-based fetch
Promio Post-delivery optimization, not traditional URL fetch
Maxemail Import at design time; snippets for send-time but internal only
EmailPlatform RSS/XML/Social feeds only, not arbitrary HTML

❌ No Support

These ESPs do not support fetching external HTML content at send time. Content must be passed via API or pre-synced.

ESP Notes
HubSpot HubL does not support external HTTP requests in email templates
SendGrid Must pass content via API payload using Handlebars
Marketo Velocity scripts cannot make external HTTP requests
Pardot Rule-based dynamic content using prospect fields only
Beehiiv No external fetch at send time
Custobar Event-based content only; all content must be in event object
Rule.io Static content defined before send, not fetched at send time
Pure360 API-based personalization; no external URL fetch
